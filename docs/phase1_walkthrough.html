3 · Schemas & Config (write these first)

Open pipeline/config.py and define two frozen dataclasses: PostgresConfig (host/port/db/user/password, plus a .dsn property) and PipelineConfig (input paths, window sizes, lateness, dedup TTL, anomaly threshold). Read every value from os.getenv with a sane default — that's the 12-factor way.

Then open pipeline/schemas.py and define two dataclasses for the upstream payloads. Hash sensitive fields at the boundary — once it enters a PCollection, raw PII must be gone.

# pipeline/schemas.py  — reference snippet
import hashlib, json, uuid
from dataclasses import dataclass, field
from datetime import datetime, timezone
from dateutil import parser as dt_parser

def _parse_ts(v: str) -> datetime:
    dt = dt_parser.isoparse(v)
    if dt.tzinfo is None: dt = dt.replace(tzinfo=timezone.utc)
    return dt.astimezone(timezone.utc)

def _sha256(v: str) -> str:
    return hashlib.sha256(v.encode("utf-8")).hexdigest()

@dataclass(frozen=True)
class ErpInvoiceEvent:
    invoice_id: str
    po_no: str
    vendor_id: str
    invoice_amount: float
    invoice_timestamp: datetime
    bank_account_hash: str               # never raw!
    upc_no: str | None = None
    email_id: str | None = None
    event_uuid: str = field(default_factory=lambda: str(uuid.uuid4()))

    @classmethod
    def from_json(cls, raw: str) -> "ErpInvoiceEvent":
        d = json.loads(raw)
        return cls(
            invoice_id        = str(d["invoice_id"]),
            po_no             = str(d["po_no"]),
            vendor_id         = str(d["vendor_id"]),
            invoice_amount    = float(d["invoice_amount"]),
            invoice_timestamp = _parse_ts(d["invoice_timestamp"]),
            bank_account_hash = _sha256(str(d.get("bank_account_details",""))),
            upc_no            = d.get("upc_no"),
            email_id          = d.get("email_id"),
        )

    @property
    def event_time(self): return self.invoice_timestamp

Do the same for WmsReceivingEvent (no PII; just typed fields). Try writing it yourself before peeking. 🐾

STEP 4.1

4.1 · Skeleton Pass-Through

Goal: prove that Beam can read your JSONL files, parse them, and print them. No fraud logic. No state. No windows. Just plumbing.

What to build

In pipeline/transforms.py add two parsing DoFns with dead-letter side outputs:

# pipeline/transforms.py — Step 4.1
import logging, apache_beam as beam
from apache_beam import pvalue
from .schemas import ErpInvoiceEvent, WmsReceivingEvent

log = logging.getLogger(__name__)

class ParseErpLine(beam.DoFn):
    DEAD_LETTER = "dead_letter"
    def process(self, line: str):
        try:
            yield ErpInvoiceEvent.from_json(line)
        except Exception as exc:
            log.warning("Bad ERP payload: %s | err=%s", line[:120], exc)
            yield pvalue.TaggedOutput(self.DEAD_LETTER, {"raw": line, "err": str(exc)})

class ParseWmsLine(beam.DoFn):
    DEAD_LETTER = "dead_letter"
    def process(self, line: str):
        try:
            yield WmsReceivingEvent.from_json(line)
        except Exception as exc:
            log.warning("Bad WMS payload: %s | err=%s", line[:120], exc)
            yield pvalue.TaggedOutput(self.DEAD_LETTER, {"raw": line, "err": str(exc)})

In pipeline/main.py wire up the simplest possible DAG:

# pipeline/main.py — Step 4.1
import logging, apache_beam as beam
from apache_beam.options.pipeline_options import PipelineOptions, StandardOptions
from .config import PipelineConfig
from .transforms import ParseErpLine, ParseWmsLine

def run():
    logging.basicConfig(level=logging.INFO,
        format="%(asctime)s [%(levelname)s] %(name)s :: %(message)s")
    cfg = PipelineConfig()

    opts = PipelineOptions()
    opts.view_as(StandardOptions).runner = "DirectRunner"

    with beam.Pipeline(options=opts) as p:
        wms = (p | "ReadWms"  >> beam.io.ReadFromText(cfg.wms_input_path)
                 | "ParseWms" >> beam.ParDo(ParseWmsLine()).with_outputs(
                       ParseWmsLine.DEAD_LETTER, main="ok"))
        erp = (p | "ReadErp"  >> beam.io.ReadFromText(cfg.erp_input_path)
                 | "ParseErp" >> beam.ParDo(ParseErpLine()).with_outputs(
                       ParseErpLine.DEAD_LETTER, main="ok"))

        wms.ok | "PrintWms" >> beam.Map(lambda e: print("WMS:", e))
        erp.ok | "PrintErp" >> beam.Map(lambda e: print("ERP:", e))

if __name__ == "__main__":
    run()

Run it

python -m pipeline.main 2>&1 | head -n 20

✅ Success criteria

  • You see WMS: WmsReceivingEvent(...) and ERP: ErpInvoiceEvent(...) lines.
  • No traceback, no exit code ≠ 0.
  • If you feed a malformed line, it shows up in a WARNING log line instead of crashing.
Common pitfalls:
  • Forgot __init__.py in pipeline/ → import errors.
  • Running python pipeline/main.py instead of python -m pipeline.main → relative-import breakage.
  • Wrong file path in cfg.erp_input_path → silent empty output.

STEP 4.2

4.2 · Stateful Deduplication

Goal: any invoice_id seen before in the last 24 hours must be silently dropped.

Why stateful and not GroupByKey?

GroupByKey only dedupes within a single window. Real retries from upstream systems land hours apart. We need per-key memory that survives across windows, with a TTL so memory stays bounded. Beam's ReadModifyWriteStateSpec + a TimerSpec is the textbook tool.

Build a tiny "duplicates only" test file first

# Pick a real line and clone it 3 times -> instant test fixture
head -n 1 data/erp_invoices.jsonl > data/erp_dups.jsonl
head -n 1 data/erp_invoices.jsonl >> data/erp_dups.jsonl
head -n 1 data/erp_invoices.jsonl >> data/erp_dups.jsonl
wc -l data/erp_dups.jsonl   # should print 3

Add the stateful DoFn

# pipeline/transforms.py — append Step 4.2
from apache_beam.transforms.userstate import (
    ReadModifyWriteStateSpec, TimerSpec, on_timer
)
from apache_beam.coders import StrUtf8Coder
from apache_beam.utils.timestamp import Duration, Timestamp

class DeduplicateInvoicesDoFn(beam.DoFn):
    DUPLICATES = "duplicates"
    SEEN_STATE   = ReadModifyWriteStateSpec("seen", StrUtf8Coder())
    EXPIRY_TIMER = TimerSpec("expiry", beam.TimeDomain.WATERMARK)

    def __init__(self, ttl_seconds: int): self._ttl = ttl_seconds

    def process(self, element,
                seen=beam.DoFn.StateParam(SEEN_STATE),
                timer=beam.DoFn.TimerParam(EXPIRY_TIMER),
                ts=beam.DoFn.TimestampParam):
        invoice_id, event = element
        if seen.read() == "1":
            yield pvalue.TaggedOutput(self.DUPLICATES, event); return
        seen.write("1")
        timer.set(Timestamp(ts.micros / 1_000_000) + Duration(self._ttl))
        yield event

    @on_timer(EXPIRY_TIMER)
    def _gc(self, seen=beam.DoFn.StateParam(SEEN_STATE)):
        seen.clear()                    # free memory after TTL

class DeduplicateInvoices(beam.PTransform):
    def __init__(self, ttl_seconds: int):
        super().__init__(); self._ttl = ttl_seconds
    def expand(self, pcoll):
        return (pcoll
            | "KeyByInvoiceId" >> beam.Map(lambda e: (e.invoice_id, e))
            | "StatefulDedup"  >> beam.ParDo(
                DeduplicateInvoicesDoFn(self._ttl)
              ).with_outputs(DeduplicateInvoicesDoFn.DUPLICATES, main="unique"))

Wire it into main.py

State requires a keyed input and lives inside the GlobalWindows by default, so re-window before dedup:

# pipeline/main.py — replace the erp.ok branch
from .transforms import DeduplicateInvoices

deduped = (erp.ok
    | "GlobalWindow"   >> beam.WindowInto(beam.window.GlobalWindows())
    | "DedupInvoices"  >> DeduplicateInvoices(ttl_seconds=24*60*60))

deduped["unique"]     | "PrintUnique" >> beam.Map(lambda e: print("UNIQ:", e.invoice_id))
deduped["duplicates"] | "PrintDup"    >> beam.Map(lambda e: print("DUP :", e.invoice_id))

Test it

ERP_INPUT_PATH=data/erp_dups.jsonl python -m pipeline.main 2>&1 | grep -E "UNIQ|DUP"
# Expected: exactly 1 UNIQ line and 2 DUP lines
Mental model: the state cell is a tiny key-value store, scoped to this DoFn instance × this key × this window. The timer fires on the watermark, not wall-clock time — that's how Beam keeps state bounded even when the upstream is firehose-fast.

STEP 4.3

4.3 · Time Elements + Sliding Window Velocity

Goal: if a vendor emits ≥3 identical invoice amounts inside a 10-minute window, raise a VELOCITY alert.

Step 1: promote payload timestamp to event time

Until you do this, Beam thinks every record arrived at processing time (= now). Your sliding window will see one giant burst instead of the actual temporal distribution. This is the #1 streaming bug. 

# pipeline/transforms.py — append Step 4.3
from apache_beam.transforms import window

class AssignEventTimestamp(beam.DoFn):
    def process(self, element):
        yield window.TimestampedValue(element, element.event_time.timestamp())

Step 2: the velocity check itself

class FlagDuplicateAmounts(beam.DoFn):
    MIN_REPEATS = 3
    def process(self, element, w=beam.DoFn.WindowParam):
        vendor_id, events = element
        buckets = {}
        for ev in events:
            key = f"{ev.invoice_amount:.2f}"
            buckets.setdefault(key, []).append(ev)
        for amt, grp in buckets.items():
            if len(grp) < self.MIN_REPEATS: continue
            for ev in grp:
                yield {
                    "invoice_id": ev.invoice_id,
                    "vendor_id":  vendor_id,
                    "rule_name":  "VELOCITY",
                    "severity":   "HIGH",
                    "reason":     f"{len(grp)}x ${amt} in 10-min window",
                    "evidence":   {"amount": float(amt),
                                   "occurrences": len(grp),
                                   "invoice_ids": [e.invoice_id for e in grp]},
                }

class VelocityFraudCheck(beam.PTransform):
    def __init__(self, window_s=600, period_s=60, lateness_s=3600):
        super().__init__()
        self._w, self._p, self._l = window_s, period_s, lateness_s
    def expand(self, pcoll):
        return (pcoll
            | "Sliding"    >> beam.WindowInto(
                window.SlidingWindows(self._w, self._p),
                allowed_lateness=self._l)
            | "KeyByVendor"   >> beam.Map(lambda e: (e.vendor_id, e))
            | "GroupByVendor" >> beam.GroupByKey()
            | "DetectVelocity" >> beam.ParDo(FlagDuplicateAmounts()))

Step 3: wire it into main.py

from .transforms import AssignEventTimestamp, VelocityFraudCheck

unique_invoices = deduped["unique"] | "AssignTs" >> beam.ParDo(AssignEventTimestamp())

velocity_alerts = unique_invoices | "Velocity" >> VelocityFraudCheck(
    window_s=600, period_s=60, lateness_s=3600)

velocity_alerts | "PrintAlerts" >> beam.Map(lambda a: print("ALERT:", a))

✅ Success criteria

  • Your mock data's V-1003 burst (6× $999.99) produces ALERT lines with "rule_name": "VELOCITY".
  • You see multiple alerts per invoice (one per overlapping window) — that's expected with sliding windows.
Why sliding (not fixed) windows? With 10-min fixed windows, a burst that straddles a window boundary would split into two halves and might miss the threshold. Sliding windows (size=10m, period=1m) guarantee every burst is fully covered by at least one window. The tradeoff is alert dedup downstream.

STEP 4.4

4.4 · Side Input Enrichment + Defensive Fallback

Goal: score every invoice against its vendor's 90-day baseline. If the baseline DB call fails OR the vendor is missing, the pipeline must not crash — it must emit a "Review Required - Fallback Mode" alert and keep processing.

Step 1: load the baseline once (with try/except)

# pipeline/transforms.py — append Step 4.4
def load_vendor_baseline(pg):
    import psycopg2
    out = {}
    try:
        with psycopg2.connect(pg.dsn) as conn, conn.cursor() as cur:
            cur.execute("""SELECT vendor_id, avg_invoice_amount,
                                  stddev_invoice_amount, p99_invoice_amount,
                                  avg_daily_invoice_cnt
                           FROM vendor_90day_baseline""")
            for vid, avg, sd, p99, daily in cur.fetchall():
                out[vid] = {"avg": float(avg), "stddev": float(sd),
                            "p99": float(p99), "daily": int(daily)}
        log.info("Loaded vendor baseline: %d vendors", len(out))
    except Exception as exc:
        # Crucial: log CRITICAL, return empty dict -> AnomalyCheck falls back
        log.critical("Baseline load FAILED — running in FALLBACK MODE: %s", exc)
    return out

Step 2: the anomaly DoFn with bulletproof error handling

class AnomalyCheckDoFn(beam.DoFn):
    FALLBACK = "Review Required - Fallback Mode"

    def __init__(self, stddev_threshold: float = 3.0):
        self._z = stddev_threshold

    def process(self, event, baseline=beam.DoFn.SideInputParam):
        try:
            stats = baseline.get(event.vendor_id)
            if stats is None:
                log.warning("No baseline for vendor=%s — FALLBACK", event.vendor_id)
                yield self._fallback(event, "No baseline available"); return

            avg, sd = stats["avg"], stats["stddev"]
            if sd <= 0:
                yield self._fallback(event, "Degenerate stddev"); return

            z = (event.invoice_amount - avg) / sd
            if abs(z) >= self._z:
                yield {
                    "invoice_id": event.invoice_id,
                    "vendor_id":  event.vendor_id,
                    "rule_name":  "ANOMALY",
                    "severity":   "CRITICAL" if abs(z) >= 5 else "HIGH",
                    "reason":     f"{z:+.2f}σ from avg ${avg:.2f}",
                    "evidence":   {"amount": event.invoice_amount,
                                   "baseline_avg": avg, "z_score": round(z,3)},
                }
        except Exception as exc:
            log.critical("Scoring crashed for invoice=%s: %s",
                         event.invoice_id, exc)
            yield self._fallback(event, f"Scoring error: {exc}")

    def _fallback(self, event, reason):
        return {
            "invoice_id": event.invoice_id,
            "vendor_id":  event.vendor_id,
            "rule_name":  "FALLBACK",
            "severity":   "MEDIUM",
            "reason":     f"{self.FALLBACK}: {reason}",
            "evidence":   {"amount": event.invoice_amount},
        }

🧪 The chaos test (this is the important part)

  1. Unknown vendor → fallback path: mock data already includes vendors V-9001 / V-9002 that aren't in the baseline table. Run the pipeline — you should see "rule_name": "FALLBACK" alerts for them, and zero crashes.
  2. DB failure → global fallback: stop Postgres and re-run: 
    docker compose stop postgres
python -m pipeline.main 2>&1 | grep -E "CRITICAL|FALLBACK" | head -n 5
docker compose start postgres
    You should see one CRITICAL log line ("Baseline load FAILED") followed by every record going to FALLBACK. No traceback. No crash.
  3. Real anomaly: the V-1005 $25,000 invoice (vs. $250 avg) should produce a "rule_name": "ANOMALY", "severity": "CRITICAL" alert.
The design principle: fail soft, alert loud. In a fraud pipeline, dropping a record silently is worse than over-flagging. Fallback alerts go to a human investigator queue — that's the safety net.

STEP 5

5 · Persist to Postgres (replace the prints)

Now swap the debug prints for real sinks. The pattern: one base class that batches inserts via psycopg2.extras.execute_values, three subclasses (bronze / silver / gold) that only differ by SQL + row projection. That's DRY at its best.

# pipeline/transforms.py — append Step 5
import json
from typing import Any, Dict, List, Tuple

class _BatchedPostgresWriter(beam.DoFn):
    BATCH_SIZE = 500
    INSERT_SQL = ""
    def __init__(self, pg): self._pg = pg; self._buf: List[Tuple] = []
    def _to_row(self, e: Dict[str, Any]) -> Tuple: raise NotImplementedError
    def setup(self):
        import psycopg2
        self._conn = psycopg2.connect(self._pg.dsn); self._conn.autocommit = False
    def teardown(self):
        if self._buf: self._flush()
        self._conn.close()
    def finish_bundle(self):
        if self._buf: self._flush()
    def process(self, element):
        self._buf.append(self._to_row(element))
        if len(self._buf) >= self.BATCH_SIZE: self._flush()
    def _flush(self):
        from psycopg2.extras import execute_values
        rows, self._buf = self._buf, []
        try:
            with self._conn.cursor() as cur:
                execute_values(cur, self.INSERT_SQL, rows, page_size=self.BATCH_SIZE)
            self._conn.commit()
        except Exception as exc:
            log.error("Postgres flush failed (%d rows): %s", len(rows), exc)
            self._conn.rollback()

class WriteGoldAlert(_BatchedPostgresWriter):
    INSERT_SQL = """INSERT INTO gold_fraud_alerts
        (invoice_id, vendor_id, rule_name, severity, reason, evidence,
         window_start, window_end) VALUES %s"""
    def _to_row(self, a):
        return (a["invoice_id"], a["vendor_id"], a["rule_name"], a["severity"],
                a["reason"], json.dumps(a["evidence"], default=str),
                a.get("window_start"), a.get("window_end"))
